# 订单追踪系统 — 需求文档

> 2026-06-10 | Next.js 16 + React 19 + TypeScript + Prisma 7 + SQLite

---

## 一、项目概述

电商订单利润追踪系统。以可编辑表格为核心，替代手动 Excel。每行一条订单，双击单元格编辑，利润自动计算，仪表盘实时汇总。

## 二、数据模型（26 字段）

**客户**：名称、地区、地址  
**货品**：名称、状态、上游对接、优先发货、是否退货、售卖状态  
**金额**：包装快递费、进价、卖价、实付  
**退货**：退货包装成本费、退货进价、退货卖价、实退  
**自动计算**：利润(=实付-进价-包装快递费)、实退利润(=实退-退货进价-退货包装成本费)  
**粉丝福利**：铁粉、钻粉、挚爱粉、超粉  
**日期备注**：开始日期(精确到分钟)、完成日期、备注

## 三、功能清单

可编辑表格（文字/数字/下拉/日期/复选框）、颜色药丸标签、斑马条纹、四列冻结、列筛选、回车搜索、退货整组搬运、利润自动计算、CSV/Excel 导出、Excel 导入、批量删除/改状态、仪表盘 KPI + 时间筛选 + 柱状图 + 预售看板、下拉选项管理、Zod 白名单校验

## 四、出错记录与规避

1. **Zod .partial() + .default() 数据清零** — 更新 schema 不能用 partial+default 组合，必须独立声明 optional 字段
2. **硬编码 CSS 偏移重叠** — 多列冻结用双表方案，不用像素估算
3. **更新时未合并已有数据算利润** — 先读数据库再合并计算
4. **异步请求竞态** — 用递增请求序号拒绝旧响应
5. **Python 生成 JSX 语法错误** — 大段 JSX 手写，不用脚本拼接
